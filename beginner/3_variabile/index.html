Variabile

Stocați valori în JavaScript

O variabilă este un container pentru stocarea valorilor. În mod tradițional, acest lucru a fost făcut cu cuvântul rezervat var, dar ES6 a introdus două cuvinte noi rezervate, const și let.

Următorul cod arată cum să creați o nouă variabilă:

let variable;

O variabilă poate stoca valori, pentru aceasta folosim operatorul de atribuire =: 

var firstVar = "Hello World";
let secondVar = "I 'm Heisenberg";
const ID_PERSON = "28934289";

console.log(firstVar);
console.log(secondVar);
console.log(ID_PERSON);

În exemplul de mai sus, șirul este stocat într-un spațiu de memorie indicând variabila firstVariable, astfel, putem accesați această valoare ori de câte ori facem referință la variabilă.


Există multe moduri de a declara o variabilă, putem face declarație și atribuire pe aceeași linie, declara multiplu variabile în același timp (deși nu este foarte recomandat), folosind stilul "comma-first"... 

// declarare și atribuire
let firstVariable = "Salut Lume";

// declarație pe aceeași linie
let prenume = 'Victoria', nume = 'Chetraru', varsta = 27;

// declararea "coma first"
let prenume = 'Victoria'
    , nume = 'Chetraru'
    , varsta = 27;

Toate aceste forme fac, teoretic, același lucru, utilizarea fiecăreia dintre aceste stiluri vor depinde de ghidul de stil de urmat pentru proiect.

Declararea variabilelor

Există două limitări importante în declararea variabilelor în JavaScript:

  1. Numele poate avea doar litere, cifre sau simboluri $ și _.
  2. Primul caracter nu poate fi o cifră.

În plus, JavaScript folosește CamelCase în declarația variabilă, așa că, deși unele nume sunt permise, acestea nu sunt recomandate de ghidul de stil.

Astfel, am avea aceste exemple valide și invalide:

// Variabile valide

let variableCamelCase;
let variable123;

let $ = 1; // Este valabil si declararea $
let _ = 1; // La fel ca și cum ai declara ca variabilă _
console.log($ + _); // Această sumă poate fi făcută

// Variabile nevalide
let invalid-variable;
let 3invalidNumber;

// Variabile valide, dar nu recomandate

let snake_case;
let allnamesinsmallcase;

Acum să vorbim despre diferențele dintre var, let și const deoarece la prima vedere par să servească aceluiași scop, dar există anumite detalii care le fac complet diferite.

Var vs Let

var și let sunt cele două cuvinte rezervate în JavaScript pentru a declara variabile.let a fost introdus în ES6 pentru a rezolva mai multe dintre confuziile generate de var cu comportamentul său, ceea ce a dus la multe erori.

Reguli de aplicare

Cea mai mare diferență are de-a face cu regulile de aplicare, vom vedea în profunzime conceptul de aplicare mai târziu, practic este mediul încapsulat în care se execută o bucată de cod. Variabilele declarate cu var sunt delimitate de corpul funcției (sfera funcției) în timp ce leteste mărginit de blocul care îl înconjoară, marcat de { } (sfera blocului) 

function scopeRules() {
    var hello = "Hello";
    let world = "World";

    console.log(hello, world); // Hello World

    {
      var name = "Andrei"
      let surname = "Moraru";
      console.log(name, surname); // Andrei Moraru
    }

    console.log(name); // Andrei
    //console.log(surname); // ReferenceError
    console.log("Reference Error");
}

Hoisting

Ridicarea este unul dintre cei mai confuzi termeni din JavaScript, practic am vorbit deja despre JavaScript ca fiind un limbaj interpretat care trece prin linie cu linie și execută ceea ce merge găsirea, dar realitatea este că merge linie cu linie de două ori programul, o dată pentru a rezerva variabile și alte lucruri, iar al doilea pentru a executa codul. Hoisting este procesul de rezervare a spațiului de memorie pentru variabile și funcții.


În JavaScript, var face variabila ridicată (inițializat înainte de a fi declarat cu nedefinit), în timp ce let va lansa un ReferenceError dacă este apelat înainte de a fi declarat. 

// hoisted
console.log(hoisted);
var hoisted = "hoisted";
console.log(hoisted)

// not hoisted
//console.log(notHoisted); ReferenceError
console.log("Reference Error");
let notHoisted = "not hoisted";
console.log(notHoisted);

Proprietate globală

În contextul global var va crea o proprietate pe obiectul global, în timp ce let nu. 

var globalVar = "Global";
let notGlobalVar = "Not global";

console.log(window.globalVar);
console.log(window.notglobalVar);

Re-Declarație

În modul strict, var permite re-declararea variabilelor în timp ce let va genera o SintaxError 

'use strict';
var redeclare = "hello";
var redeclare = "world"; 

let notRedeclare = "hello"; 
//let notRedeclare = "bar2"; 
console.log("Sintax Error");

Const

Constantele sunt variabile care nu își pot schimba valoarea atunci când sunt atribuite. Acest tip de declarație este foarte util atunci când valoarea atribuită nu trebuie modificată în nicio circumstanță în timpul execuției, cum ar fi constante matematice, culori sau alte valori fixe. 

const PI = 3.14;
const COLOR_BLUE = "00F";
const COUNTRY_CODE_MOLDOVA = "MD";

Ghid stilistic

Deși variabilele din JavaScript urmează stilul CamelCase, utilizarea majusculelor pentru declararea constantelor este larg răspândită, folosind litere mici pentru a separa cuvintele. Puteți consulta stilul declarației variabilelor în Ghid de stil MDN.
